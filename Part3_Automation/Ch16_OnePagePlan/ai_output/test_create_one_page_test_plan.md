# 一頁式測試計畫（One-Page Test Plan）
**系統：會員折扣（運費）系統｜發佈時間：明天中午**

| 欄位 | 內容                                                                                                                                |
|---|-----------------------------------------------------------------------------------------------------------------------------------|
| **Context and Strategy** | 明日中午上線的高風險功能變更，核心策略採 **Risk-based Testing**，集中驗證「第三方登入後會員身分 × 運費規則 × 金額邊界值」的關鍵路徑，以快速回饋與可退回版本為優先。                                  |
| **In Scope** | - Google / Apple ID 登入後會員等級判定（VIP / 一般）<br>- VIP 運費免運規則（>2000）<br>- 一般會員固定運費 100 元<br>- 金額邊界值（1999 / 2000 / 2001）<br>- 前後端運費計算一致性 |
| **Out of Scope** | - 非活動期間運費規則<br>- 其他優惠／折扣邏輯<br>- 會員升級／降級流程<br>- 非運費相關 UI 美化與文案                                                                     |
| **Risks** | **最高風險：VIP 身分或金額判斷錯誤導致運費誤收／少收**。後果包含：<br>- 使用者被錯誤收取運費 → 投訴與信任下降<br>- 大量訂單少收運費 → 直接營收損失<br>- 前後端不一致 → 客服與帳務對帳成本暴增                  |
| **Resource / Timeline** | - 測試人力：1 位測試人員<br>- 時間：發佈前 1 天（今天）<br>- 執行方式：手動關鍵流程測試＋必要 API 驗證<br>- 發佈前提供 Go / No-Go 建議                                          |
| **Environment and Tools** | - Staging 環境（與 Production 同設定）<br>- 測試帳號：VIP / 一般會員各 1 組（含第三方登入）<br>- 工具：瀏覽器 DevTools、API 測試工具（Postman / curl）、簡易檢查清單             |
